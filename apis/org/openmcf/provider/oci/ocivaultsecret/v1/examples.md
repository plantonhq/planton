# OciVaultSecret Examples

## Explicit Base64 Content

A secret with user-provided base64-encoded content — suitable for a database password or API key:

```yaml
apiVersion: oci.openmcf.org/v1
kind: OciVaultSecret
metadata:
  name: db-password
  labels:
    openmcf.org/provisioner: pulumi
    pulumi.openmcf.org/organization: my-org
    pulumi.openmcf.org/project: my-project
    pulumi.openmcf.org/stack.name: dev.OciVaultSecret.db-password
spec:
  compartmentId:
    value: "ocid1.compartment.oc1..example"
  secretName: "dev-db-password"
  vaultId:
    value: "ocid1.vault.oc1..example"
  keyId:
    value: "ocid1.key.oc1..example"
  secretContent:
    content: "c2VjcmV0LXZhbHVl"
```

## Auto-Generated Passphrase

OCI generates a random passphrase using a predefined template. The secret is encrypted by a KMS key referenced via `valueFrom`:

```yaml
apiVersion: oci.openmcf.org/v1
kind: OciVaultSecret
metadata:
  name: app-passphrase
  labels:
    openmcf.org/provisioner: pulumi
    pulumi.openmcf.org/organization: my-org
    pulumi.openmcf.org/project: my-project
    pulumi.openmcf.org/stack.name: prod.OciVaultSecret.app-passphrase
spec:
  compartmentId:
    valueFrom:
      kind: OciCompartment
      name: prod-compartment
      fieldPath: status.outputs.compartmentId
  secretName: "app-passphrase"
  vaultId:
    valueFrom:
      kind: OciKmsVault
      name: prod-vault
      fieldPath: status.outputs.vaultId
  keyId:
    valueFrom:
      kind: OciKmsKey
      name: prod-key
      fieldPath: status.outputs.keyId
  enableAutoGeneration: true
  secretGenerationContext:
    generationType: passphrase
    generationTemplate: "SECRET_TEMPLATE_DBAAS"
    passphraseLength: 32
```

## Expiry and Reuse Rules

A secret with a 30-day version expiry (content retrieval blocked after expiry) and a reuse prevention rule:

```yaml
apiVersion: oci.openmcf.org/v1
kind: OciVaultSecret
metadata:
  name: rotating-credential
  labels:
    openmcf.org/provisioner: pulumi
    pulumi.openmcf.org/organization: my-org
    pulumi.openmcf.org/project: my-project
    pulumi.openmcf.org/stack.name: prod.OciVaultSecret.rotating-credential
spec:
  compartmentId:
    value: "ocid1.compartment.oc1..example"
  secretName: "service-credential"
  vaultId:
    value: "ocid1.vault.oc1..example"
  keyId:
    value: "ocid1.key.oc1..example"
  secretContent:
    content: "c2VjcmV0LXZhbHVl"
  secretRules:
    - ruleType: secret_expiry_rule
      secretVersionExpiryInterval: "P30D"
      isSecretContentRetrievalBlockedOnExpiry: true
    - ruleType: secret_reuse_rule
      isEnforcedOnDeletedSecretVersions: true
```

## Scheduled Rotation Against Autonomous Database

A credential that automatically rotates every 30 days. OCI generates a new passphrase and updates the Autonomous Database admin password:

```yaml
apiVersion: oci.openmcf.org/v1
kind: OciVaultSecret
metadata:
  name: adb-credential
  labels:
    openmcf.org/provisioner: pulumi
    pulumi.openmcf.org/organization: my-org
    pulumi.openmcf.org/project: my-project
    pulumi.openmcf.org/stack.name: prod.OciVaultSecret.adb-credential
spec:
  compartmentId:
    value: "ocid1.compartment.oc1..example"
  secretName: "adb-admin-credential"
  vaultId:
    value: "ocid1.vault.oc1..example"
  keyId:
    value: "ocid1.key.oc1..example"
  enableAutoGeneration: true
  secretGenerationContext:
    generationType: passphrase
    generationTemplate: "SECRET_TEMPLATE_DBAAS"
    passphraseLength: 24
  rotationConfig:
    isScheduledRotationEnabled: true
    rotationInterval: "P30D"
    targetSystemDetails:
      targetSystemType: adb
      adbId:
        valueFrom:
          kind: OciAutonomousDatabase
          name: prod-adb
          fieldPath: status.outputs.autonomousDatabaseId
```

## Auto-Generated SSH Key with Metadata

An SSH key pair generated by OCI, with custom metadata for administrative tracking:

```yaml
apiVersion: oci.openmcf.org/v1
kind: OciVaultSecret
metadata:
  name: ssh-key
  labels:
    openmcf.org/provisioner: pulumi
    pulumi.openmcf.org/organization: my-org
    pulumi.openmcf.org/project: my-project
    pulumi.openmcf.org/stack.name: prod.OciVaultSecret.ssh-key
spec:
  compartmentId:
    value: "ocid1.compartment.oc1..example"
  secretName: "bastion-ssh-key"
  vaultId:
    value: "ocid1.vault.oc1..example"
  keyId:
    value: "ocid1.key.oc1..example"
  description: "SSH key pair for bastion access"
  enableAutoGeneration: true
  secretGenerationContext:
    generationType: ssh_key
    generationTemplate: "2048"
  secretMetadata:
    purpose: "bastion-access"
    team: "platform"
```

## Common Operations

### Update secret content

Change the `secretContent.content` field to the new base64-encoded value and re-apply. OCI creates a new secret version automatically. The previous version remains accessible until it expires (if an expiry rule is configured).

### Switch from explicit to auto-generated content

Remove the `secretContent` block, set `enableAutoGeneration: true`, and add a `secretGenerationContext` block. Re-apply the manifest. Note: this may require secret recreation depending on OCI API behavior.

### Add an expiry rule to an existing secret

Append a new entry to the `secretRules` list with `ruleType: secret_expiry_rule` and the desired interval, then re-apply.

## Best Practices

1. **Use auto-generation for machine credentials** — avoids storing secrets in manifests or CI pipelines.
2. **Use expiry rules for compliance** — ensures secret versions do not remain valid indefinitely.
3. **Use scheduled rotation for database credentials** — reduces the blast radius of compromised credentials.
4. **Use `valueFrom` references** for `vaultId` and `keyId` — avoids hardcoding OCIDs and ensures correct dependency ordering.
5. **Set `isSecretContentRetrievalBlockedOnExpiry: true`** for sensitive secrets — prevents retrieval of expired content.
6. **Use `secretMetadata` for administrative context** — helps teams identify secret purpose and ownership.
