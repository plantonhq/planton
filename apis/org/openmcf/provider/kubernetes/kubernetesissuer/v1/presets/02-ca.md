# CA Issuer

This preset creates a CA Issuer that signs certificates using a CA keypair stored in a Kubernetes Secret. The Secret must contain `tls.crt` (CA certificate) and `tls.key` (CA private key), and must reside in the same namespace as the Issuer.

## When to Use

- You have a CA keypair (typically generated by a KubernetesCertificate with `is_ca=true`)
- You need to sign leaf certificates for internal services using your own CA
- You're building an internal PKI where certificates are not externally trusted

## Prerequisites

- The CA Secret must already exist in the target namespace
- Typically, the CA Secret is created by deploying a KubernetesCertificate with `is_ca=true` that references a SelfSigned Issuer (preset 01-self-signed)

## Placeholders to Replace

| Placeholder | Description | Where to Find |
|---|---|---|
| `<your-namespace>` | Namespace where the Issuer and CA Secret live | Your cluster namespace list |
| `<your-ca-secret-name>` | Name of the Kubernetes Secret containing the CA keypair | KubernetesCertificate outputs (`secret_name`) or `kubectl get secrets -n <namespace>` |

## Related Presets

- **01-self-signed** -- Use to bootstrap the CA Secret via a SelfSigned Issuer + CA Certificate
