# `src/site` — site config & domain data

The single source of truth for everything planton.dev needs to *say* and *link
to*. Presentation components (`src/components/**`) import from here and never
hardcode a URL, count, command, or brand asset path.

| File | Owns |
|---|---|
| `nav.ts` | External URLs (GitHub, Discord, planton.ai, Charts) and header/footer link lists. |
| `download.ts` | The `/download` CTA target, supported OS list, and CLI install methods. |
| `providers.ts` | Provider/tool logos (each a real asset in `public/images/providers`). |
| `stats.ts` | Display-ready numbers derived from `stats.generated.ts`. |
| `stats.generated.ts` | GENERATED by `scripts/generate-stats.ts` at build — do not edit. |

## Rules

- No hardcoded links/counts/commands in components — add them here.
- Brand logos must reference real files in `public/`. Never invent a mark.
- Numbers come from `stats` (code-derived), never a typed-in literal.

Import via the barrel: `import { stats, DEPLOY_CLOUDS, DOWNLOAD_HREF } from "@/site";`
